Q: When I insert the CD in the drive, the installation doesn't start automatically. What's wrong?

R: You probably deactivated the auto-insert notification. To enable this notification, open the Start menu, select Settings, then Control Panel, and double-click on the System icon. Double-click on the CDROM branch on the Device Manager tab, and then double-click the entry for your CD-ROM drive. On the Settings tab, check the Auto Insert Notification check box. Click OK, click Close, and then click Yes when you are prompted to restart your computer.

You can also launch manually the auto-insert application. Open your Windows Explorer. Double-click on your CD-ROM drive. Click on the AUTORUN directory, then double-click on the AUTORUN.EXE program.

Q: When I insert the CD, the following error is displayed: "c:\windows\system32\autoexec.nt. The system file is not suitable for running MS-DOS and Microsoft applications. Choose 'Close' to terminate the application". The name of the file may be "autoexec.nt" or "config.nt"

R: Please refer to the following article from the Microsoft Support Knowledge base:
http://support.microsoft.com/default.aspx?scid=kb;en-us;324767

When you insert the CD in the drive, Windows launches a program called AUTORUN.EXE located in the AUTORUN folder of the CD. This program is a 16-bits application that lets you choose between installing the software and visit our internet web site.

To install eMedia Card Designer, browse the "\eMedia" folder and launch the setup.exe program located in it.

Q: After installing DCOM for Windows 95 or DCOM for Windows 98, the computer asks me to reboot the system. When the system restarts, a message box tells me that SETUP.EXE cannot be found. What must I do?

R: The eMedia setup program is probably on a network drive, or on a drive that is not available at boot time. Click OK to close the box. Let your system finish the restart, then launch manually the eMedia setup program (see above). The eMedia setup program will check your system and find that DCOM is installed. The installation process will then continue.

Q: I had installed successfully DCOM, then MDAC without restarting the system. I restarted the system at the very end of the installation process. Now, eMedia doesn't work and displays error messages when I start it. Is there a bug in the installation application?

R: MDAC must be installed only once DCOM is fully functionnal. If you didn't reboot between the DCOM and MDAC installations, MDAC is incorrectly installed, and eMedia encounters problems, due to the unsuccessful installation of MDAC. Re-launch the eMedia setup program to correct the problem (see above).

Q: I installed sucessfully DCOM but didn't restart the computer. Then, I restarted the eMedia setup program, and this one tells me that DCOM is not installed!

R: This is normal. DCOM is only fully installed once your computer has been restarted.

Q: When eMedia setup installs MDAC, my computer hangs.

R: Try to use the NumLock key. If the light goes out and in, your computer isn't halted. The MDAC installation process can take a very long time. Up to ten minutes. Let MDAC installs itself completely. It can be very time consuming.

Q: I had installed successfully eMedia, but when I start the software, the following error occurs: "Error nn, sub-error 2 (EVPB1)". The eMedia.log file in the eMedia installation folder contains the following line: "EVPB1 nn : Sub-error #2". I can't start eMedia anymore.

R: This error occurs on certain systems and is due to a partial installation of MDAC components on eMedia setup. The major cause of this error is a compatibility problem between MDAC 2.5 provided with eMedia and the MDAC version currently on the system.

You can resolve this problem by installing MDAC 2.7.0.7713.4, provided on the distribution CD:

  • Insert the eMedia installation CD in your drive.
  • With your Windows Explorer, browse the "\eMedia\Support" folder.
  • Double-click on MDAC_TYP_27.EXE file and install it.

Q: I can't see the online help !

The online help uses the new HTML help system from Microsoft Corporation. If you can't see it, it can be due to a wrong installation of the HTML help system.

You can resolve this by installing the help system upgrade, Hhupd.exe, provided on the CD:

  • Insert the eMedia installation CD in your drive.
  • With your Windows Explorer, browse the "\eMedia\Support" folder.
  • Double-click on HHUPD.EXE file and install it.

If this doesn't correct your problem, please contact the eMedia support team: support@emedia-cards.com.

Q: Errors 1931 on hhctrl.ocx or hh.exe

This error can be fired by the product installation if these files are reserved by the Operating System. Discard these errors and continue installation. If you can't see the online help during program execution, please refer to the question above.

Q: All the answers above can't apply to me. I still have problems.

R: There are three ways to install eMedia manually:

  • You can use the verbose version of Setup.exe to help you to find where the problem is located,
  • You can install the packages by yourself,
  • You can install all the files manually.

1. Using the verbose version of Setup.exe

  • Copy the \eMedia folder and all its subfolders from the installation CD into a blank directory of your hard drive.
  • Delete the Setup.exe file in your target folder.
  • Rename the Setup.dat file and call it Setup.exe
  • Launch the Setup.exe program. This installation program is identical to the one provided, but displays messages about what it is doing. If errors occurs, the corresponding messages are displayed.

2. Install the packages by yourself

  • DCOM manual installation: you will find in the eMedia\Win9x directory of the CD the files DCOM95.EXE and DCOM98.EXE. Launch DCOM95.EXE if you have a Windows 95™ operating system. Launch DCOM98.EXE if you have a Windows 98™, or Windows 98SE™ operating system. Check for the error messages that could appear, and then contact the Microsoft support. If the installer ask you to reboot at the end of the process, restart your computer. If your computer runs Windows ME, Windows NT 4 or Windows 2000 operating systems, do not install DCOM.
  • MDAC manual installation: you will find in the eMedia\Common directory of the CD the file MDAC_TYP.EXE. Launch this program and check for the error messages that could appear, and then contact the Microsoft support. If the installer ask you to reboot at the end of the process, restart your computer.
  • Install Microsoft Windows Installer™ manually. You will find in the eMedia\Setup folder of the CD the files INSTMSIA.EXE. and INSTMSIW.EXE. Launch INSTMSIA.EXE if your computer runs Windows 95™, Windows 98™, Windows 98SE™ or Windows ME™ operating systems, and INSTMSIW.EXEif your computer runs Windows NT4™, Windows 2000™ or Windows XP™ operating systems. Check for the error messages that could appear and contact the Microsoft support.
  • If you plan to use DirectShot, install the support files manually. You will find in the eMedia\DirectShot directory of the CD the file DCGenAX.exe. Launch this file and follow the instructions. Check for the error messages that could appear and contact the eMedia support team.
  • eMedia manual installation: you will find in the eMedia\Setup folder of the CD the file Setup.exe. Double-click on this file to install the software. Check for the error messages that could appear then contact the eMedia support team.

3. Installing the files manually

  • You can contact us for the description of a flat installation of eMedia: support@emedia-cards.com.

 

If no problem seems to appear but the installation fails, please contact the eMedia support team: support@emedia-cards.com.
